# Consultation Flow Implementation Specification

The consultation flow system efficiently processes high user volumes (100,000+ users) through a streamlined decision tree with 3-5 targeted questions that rapidly classify users into one of twelve distinct profile types. Each profile represents a specific combination of primary interests (aesthetic preferences), secondary interests (specific goals), and tertiary interests (detailed requirements). The system captures these classifications as structured data props that are passed directly to the landing page builder, which then dynamically constructs a personalized experience matching the identified profile. This implementation focuses on performance optimization, maintaining sub-500ms response times even under heavy load, while ensuring the classification algorithm accurately maps user responses to the correct profile through weighted scoring of response patterns rather than rigid paths. The system stores minimal session data during the consultation process and emphasizes stateless operations where possible, with the final profile classification being the primary persistent data point that drives subsequent personalization throughout the user journey.
